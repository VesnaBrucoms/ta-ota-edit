Commands

The initial mission field for schema added units is where a lot of the mission making happens. A unit's initial mission is a list of simple commands that it will try to follow; with each command is split by a comma. For example:

m 546 213, w 20, a ARMCOM

The unit will move to coordinates (546, 213) then wait 20 seconds before attacking the nearest Arm Commander.

The available commands are as follows:

Command Description
m X Y Move to coordinates X and Y
p X Y Patrol between the unit's starting location and the coords given in the command.
a UNITNAME Attack the nearest specified unit type. UNITNAME can be the internal name for a unit type, or it can be a specific unit's identifier.
w SECONDS Wait the specified number of seconds before continuing with the commands. It's also possible to input decimals like 0.4.
b UNITTYPE X Y Mobile construction units only. Tells the unit to build the specified unit type at the given coordinates.
b UNITTYPE N Construction structures only. Build N number of units.
d Destroys the unit.
s Make the unit selectable while it's still performing its commands.
i UNITNAME Load inside the specified transport.
u X Y Unload the transport at specified coordinates.
wa Wait until attacked.
wa IDENTIFIER Wait until the specified unit is attacked. Unlike previous commands, this requires a unique identifier and will not accept a unit type name.
